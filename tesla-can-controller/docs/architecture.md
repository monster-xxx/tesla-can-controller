# 系统架构设计

## 🏗️ 整体架构

### 系统组成
本系统由三个主要设备组成，通过加密BLE连接协同工作：

| 设备 | 代号 | 核心功能 | 主要组件 |
|------|------|----------|----------|
| **主控制器** | 设备A | 用户界面、系统控制 | M5Dial (ESP32-S3)、LVGL、旋转编码器 |
| **CAN读取器** | 设备B | 车辆通信、数据处理 | ESP32-C6、SN65HVD230×2、FreeRTOS |
| **氛围灯控制器** | 设备C | 灯光效果、氛围控制 | ESP32、WS2812驱动、12V电平转换 |

## 📡 通信协议设计

### BLE通信协议
#### GATT服务设计
```
服务UUID: 6E400001-B5A3-F393-E0A9-E50E24DCCA9E
├── 特性UUID_RX: 6E400002-B5A3-F393-E0A9-E50E24DCCA9E
│   └── 设备A→设备B/C: 控制命令、配置数据
├── 特性UUID_TX: 6E400003-B5A3-F393-E0A9-E50E24DCCA9E
│   └── 设备B/C→设备A: 状态数据、响应确认
└── 特性UUID_NOTIFY: 6E400004-B5A3-F393-E0A9-E50E24DCCA9E
    └── 实时通知、事件推送
```

#### 数据包格式
```c
typedef struct __attribute__((packed)) {
    uint8_t header;          // 0xAA (起始符)
    uint8_t version;         // 协议版本 (0x01)
    uint8_t device_id;       // 设备ID (A=0x01, B=0x02, C=0x03)
    uint8_t message_type;    // 消息类型
    uint16_t sequence;       // 序列号
    uint16_t data_length;    // 数据长度
    uint8_t data[248];       // 数据载荷
    uint8_t checksum;        // 校验和
    uint8_t footer;          // 0x55 (结束符)
} ble_packet_t;
```

#### 加密方案
```
加密流程:
1. 密钥交换: ECDH (P-256曲线)
2. 会话密钥: HKDF-SHA256 派生
3. 数据加密: AES-256-GCM
4. 认证标签: 128-bit GCM标签

安全特性:
- 前向保密 (每次会话新密钥)
- 防重放攻击 (序列号+时间戳)
- 完整性保护 (GCM认证标签)
- 密钥轮换 (24小时自动更新)
```

# 电池状态 (ID: 0x352)

 SG_ BatteryVoltage : 7|16@1+ (0.1,0) [0|6553.5] "V" VCU,IC
 SG_ BatteryCurrent : 23|16@1- (0.1,-3276.8) [-3276.8|3276.7] "A" VCU,IC
 SG_ SOC : 39|8@1+ (0.5,0) [0|127.5] "%" VCU,IC

# 挡位信息 (ID: 0x108)

   VAL_ 0 Park 1 Reverse 2 Neutral 3 Drive 4 Sport 5 Low 6 Manual 7 Invalid

#### CAN消息处理流程
```
1. 原始CAN消息接收 (ESP32 CAN控制器)
2. 消息过滤 (ID白名单)
3. DBC解析 (信号提取)
4. 数据预处理 (单位转换、范围检查)
5. 数据聚合 (多消息关联)
6. 状态机更新 (车辆状态)
7. BLE封装传输 (加密压缩)
```

## 🔄 数据流设计

### 车辆数据采集流
```
1. 车辆传感器 → CAN总线 → 原始CAN消息
2. 设备B接收 → DBC解析 → 结构化信号
3. 信号处理 → 状态更新 → 车辆状态对象
4. 状态压缩 → BLE加密 → 发送到设备A
5. 设备A接收 → 解密解析 → UI更新
6. 用户查看 → 实时显示 → 交互反馈
```

### 控制命令执行流
```
1. 用户操作 → 设备A UI → 生成控制命令
2. 命令验证 → 权限检查 → 安全确认
3. 命令封装 → BLE加密 → 发送到设备B
4. 设备B接收 → 解密验证 → CAN命令转换
5. CAN发送 → 车辆执行 → 响应返回
6. 执行结果 → BLE返回 → UI状态更新
```

### 氛围灯控制流
```
1. 用户选择 → 设备A UI → 灯光设置
2. 设置封装 → BLE加密 → 发送到设备C
3. 设备C接收 → 解密解析 → 灯光参数
4. 效果计算 → PWM/RMT → WS2812驱动
5. 灯光变化 → 实时反馈 → 状态同步
6. 状态返回 → 设备A UI → 确认显示
```
