## 🔌 设备A: M5Dial 主控制器

### 技术规格
```
核心处理器: ESP32-S3R8 (双核Xtensa LX7, 240MHz)
内存配置: 8MB PSRAM, 16MB Flash
显示屏幕: 2.0英寸 IPS LCD (320×240)
触摸技术: 电容式多点触摸
输入设备: 旋转编码器 + 按键
振动反馈: 线性振动电机
无线通信: WiFi 4 + Bluetooth 5.0 (LE)
存储扩展: MicroSD卡槽 (最大32GB)
电源输入: USB-C (5V/2A) 或 锂电池 (3.7V)
接口: USB-C (数据/供电), Grove接口
尺寸: 66×66×20mm
重量: 约65g
```

### 电气特性
| 参数 | 最小值 | 典型值 | 最大值 | 单位 |
|------|--------|--------|--------|------|
| 工作电压 | 3.0 | 3.3 | 3.6 | V |
| 工作电流 | 80 | 120 | 200 | mA |
| 峰值电流 | - | 300 | 500 | mA |
| 工作温度 | -20 | 25 | 70 | °C |
| 存储温度 | -40 | - | 85 | °C |

### 连接器定义
```
J1: USB-C连接器
├── VBUS (5V电源输入)
├── D+ (USB数据+)
├── D- (USB数据-)
├── CC1/CC2 (配置通道)
└── GND (电源地)

J2: Grove连接器 (I2C)
├── 3V3 (3.3V输出)
├── GND
├── SDA (I2C数据)
└── SCL (I2C时钟)

J3: 电池连接器
├── BAT+ (锂电池正极)
└── BAT- (锂电池负极)
```

## 🔌 设备B: ESP32-C6 CAN读取器

### 核心模块
```
处理器: ESP32-C6 (RISC-V单核 + 802.15.4)
时钟频率: 160MHz
内存: 320KB SRAM, 4MB Flash
无线: WiFi 6 + Bluetooth 5.0 + Zigbee 3.0
CAN接口: 集成TWAI控制器
GPIO: 22个可编程引脚
工作电压: 3.0V - 3.6V
封装: QFN40 (5×5mm)
```

### CAN接口电路
```
SN65HVD230 CAN收发器:
├── 供电电压: 3.3V
├── 波特率: 最高1Mbps
├── 接口: CANH, CANL, VCC, GND, STB, RXD, TXD
├── 保护功能: 过热关断、短路保护
└── ESD保护: ±8kV接触放电
```

### 电源电路设计
```
输入: 车辆12V电源
├── 保险丝: 2A快熔保险丝
├── 反接保护: 防反接二极管
├── 滤波电路: LC滤波器
├── 第一级降压: 12V→5V (LM2596)
│   ├── 效率: >85%
│   ├── 最大电流: 3A
│   └── 纹波: <50mV
└── 第二级降压: 5V→3.3V (AMS1117)
    ├── 效率: >80%
    ├── 最大电流: 1A
    └── 纹波: <30mV
```

### PCB设计要点
```
层数: 2层PCB
尺寸: 60×40mm
材质: FR-4, 1.6mm厚度
表面处理: 沉金
特殊要求:
├── CAN走线等长匹配

├── 信号完整性考虑
└── EMC/EMI防护
```

## 🔌 设备C: ESP32 氛围灯控制器

### 核心模块
```
处理器: ESP32-WROOM-32E
时钟频率: 240MHz (双核)
内存: 520KB SRAM, 4MB Flash
无线: WiFi 4 + Bluetooth 4.2
PWM输出: 16通道LEDC
RMT: 8通道 (WS2812驱动)
GPIO: 34个可编程引脚
工作电压: 3.0V - 3.6V
封装: 38-pin邮票孔
```

### WS2812驱动电路
```
WS2812B RGB LED:
├── 工作电压: 5V (12V兼容版本)
├── 数据协议: 单线归零码
├── 数据传输率: 800Kbps
├── 刷新率: >30fps (60LEDs)
├── 灰度等级: 256级
└── 颜色深度: 24位 (8位每通道)

电平转换电路:
├── 转换芯片: 74HC245
├── 输入电压: 3.3V (ESP32)
├── 输出电压: 12V (车辆)
├── 驱动能力: 24mA per channel
└── 传输延迟: <10ns
```

### 电源管理
```
主电源: 车辆12V
├── 输入保护: 自恢复保险丝 (3A)
├── 过压保护: TVS二极管 (18V)
├── 滤波电路: π型滤波器
└── 稳压输出:
    ├── 12V输出: 直接供电 (灯带)
    ├── 5V输出: 降压模块 (逻辑电路)
    └── 3.3V输出: LDO稳压器 (ESP32)

功率计算:
├── WS2812灯带: 0.3W/LED × 60LEDs = 18W
├── ESP32模块: 0.5W
├── 逻辑电路: 0.2W
└── 总功率: ~20W (峰值)
```

设备C ↔ 灯带 (有线):

## 🔋 电源系统设计

### 整体电源架构
```
车辆12V电池
├── 主保险丝盒 (车辆原装)
├── 定制保险丝 (2A)
├── 电源分配板
│   ├── 设备B电源电路
│   ├── 设备C电源电路
│   └── 备用电源接口
└── 地线回路
    ├── 单点接地设计
    └── 低阻抗接地路径
```

### 电源时序要求
```
上电顺序:
1. 设备B先上电 (CAN读取器)
2. 设备C后上电 (氛围灯控制器)
3. 设备A最后上电 (主控制器)

断电顺序:
1. 设备A先关机
2. 设备C后关机
3. 设备B最后关机

电源监控:
├── 电压监测: 12V输入电压
├── 电流监测: 各支路电流
├── 温度监测: 电源芯片温度
└── 故障保护: 过压/过流/过热
```

### 低功耗设计
```
休眠模式:
├── 设备A: 深度睡眠 (<10μA)
├── 设备B: 轻度睡眠 (<500μA)
├── 设备C: 深度睡眠 (<50μA)
└── 唤醒方式:
    ├── 设备A: 按键唤醒
    ├── 设备B: CAN活动唤醒
    ├── 设备C: BLE连接唤醒

功耗估算 (正常工作):
├── 设备A: 120mA @ 3.3V = 0.4W
├── 设备B: 80mA @ 3.3V = 0.26W
├── 设备C: 150mA @ 3.3V = 0.5W (不含灯带)
└── 灯带: 1.5A @ 12V = 18W (最大亮度)
```

## ⚠️ 注意事项

### 安全警告
1. **车辆电气安全**
   - 断开车辆电池负极再进行连接
   - 使用合适规格的保险丝
   - 避免短路到车辆金属部件

2. **静电防护**
   - 使用防静电手环
   - 在防静电工作台操作
   - 存储时使用防静电袋

3. **操作安全**
   - 不要在车辆行驶中安装/调试
   - 避免在潮湿环境操作
   - 遵循正确的操作顺序

### 维护建议
1. **定期检查**
   - 每月检查连接是否松动
   - 检查保险丝状态
   - 清洁连接器触点

2. **故障排除**
   - 参考故障排除指南
   - 使用诊断工具
   - 联系技术支持

3. **升级更新**
   - 定期检查固件更新
   - 备份配置文件
   - 测试新功能后再部署
