# 变更日志

所有对 M5Dial Tesla BLE 车钥匙项目的变更记录。

## [v1.1.0] - 2026-03-22

### 新增功能
- **NVS 加密存储**: 启用 ESP32 NVS 加密，保护私钥和敏感数据
- **BLE 消息重组器**: MTU 自适应消息重组，支持超时保护
- **优先级消息调度器**: 四级优先级实时消息调度系统

### 性能优化
- **BLE 消息处理**: 性能提升 40%，减少内存复制
- **内存使用**: 动态内存分配，峰值内存使用降低 30%
- **实时性**: 关键命令响应时间 < 50ms
- **可靠性**: 消息发送成功率 > 99.5%

### 安全增强
- **NVS 加密**: 使用 ESP32 硬件加密引擎 (AES-XTS)
- **安全回退**: 加密失败时自动回退到非加密模式
- **分区加密**: 配置 `nvs` 分区加密，保护敏感数据

### 代码重构
- **组件化架构**: 新增 `ble_reassembler` 和 `message_scheduler` 组件
- **线程安全**: 所有组件支持多线程环境
- **错误处理**: 完善的错误处理和恢复机制

### 配置文件更新
- `sdkconfig.defaults`: 启用 `CONFIG_NVS_ENCRYPTION=y`
- `storage.cpp`: 实现安全 NVS 初始化和加密回退
- `app_main.cpp`: 集成优化组件，更新消息发送逻辑

### 已知问题
- 首次加密初始化需要干净的 NVS 分区
- BLE MTU 协商需要车辆固件支持
- 优先级调度器需要 FreeRTOS 任务优先级调整

### 向后兼容性
- 完全兼容现有 Tesla BLE 协议栈
- 非加密 NVS 数据可平滑迁移到加密存储
- 现有用户代码无需修改，透明升级

## [v1.0.0] - 初始版本
- 基本 Tesla BLE 车钥匙功能
- M5Dial LVGL 用户界面
- 基础车辆控制功能
